Linux进程及作业管理

发表于 2018-03-06 | 分类于 系统运维 |
字数统计: 2,032 | 阅读时长 ≈ 8

进程相关概念

  • 进程：
    • 处于运行状态的程序副本
    • 正在执行的程序代码的实时结果
    • 处于执行期的程序以及相关资源的总称

  • 线程：

    • 在进程中活动的对象
    • 每个线程都拥有一个独立的程序计数器、进程栈和一组进程寄存器
    • 是内核调度的对象

  • 同一个进程中的线程之间可以共享虚拟内存，但每个线程都拥有各自的虚拟处理器。

  • Windows和Solaris系统都在内核中提供了专门支持线程的机制，这些系统常常把线程称作轻量级进程，相较于重量级的进程，线程被抽象成一种耗费较少资源，运行迅速的执行单元。而对于Linux来说，它只是一种进程间共享资源的手段。
  • task struct：Linux内核存储进程信息的固定格式
  • task list：多个任务的task struct组成的链表
  • 进程创建
    • init
    • 进程都由其父进程创建
      • fork()
      • clone()
  • 进程的调度
    • 进程优先级
      • 0-139：
        • 1-99：实时优先级
        • 100-139：静态优先级
          • 数字越小，优先级越高
      • Nice值：
        • -20，19
    • 进程内存
      • Page Frame：页框，用于存储页面数据
      • 虚拟内存实现机制
        • 线性地址空间：每个进程都认为自己有3G内存可用
        • 物理地址空间：处于内存不同位置的空闲的页，由内核映射为线性地址空间
      • MMU：Memory Management Unit
      • 缺页异常：进程访问的数据不存在，向内核申请从磁盘载入
      • 常驻内存集
      • 虚拟内存集：可被交换出去的内存集
    • 进程间通信：IPC-Inter Process Communication
      • 同一主机：
        • signal
        • shm：shared memory
        • semerphor
      • 不同主机：
        • rpc：remote procecure call
        • socket
    • 进程类型
      • 守护进程：daemon，在系统引导过程中启动的进程，跟终端无关的进程
      • 前台进程：和终端相关，通过终端启动的进程
      • 注意：也可以把前台启动的进程送往后台，以守护模式运行
    • 进程状态
      • 运行态：running
      • 就绪态：ready
      • 睡眠态
        • 可中断：interruptable
        • 不可中断：uninterruptable
      • 停止态：stopped，暂停于内存中，但不会被调度，除非被手动启动
      • 僵死态：zombie
    • 进程的分类：
      • CPU-Bound：一般属于非交互式进程
      • IO-Bound：一般属于交互式进程
    • 推荐两本Linux内核的书：《Linux内核设计与实现》《深入理解Linux内核》
    • Linux内核：抢占式多任务
    • IO过程
      1. 从磁盘中把数据加载至内核内存
      2. 把数据从内核内存复制到进程内存中去

进程及作业管理

进程管理命令

  • pstree
  • ps： process state
    • Linux系统各进程的相关信息均保存在/proc/PID目录下的个文件中，进程状态查看工具即通过读取此类文件显示进程相关状态。
    • 常用组合
      • aux：
        • u：以用户为中心组织进程状态信息显示
        • a：与终端相关的进程
        • x：与终端无关的进程
        • 字段含义：
          • VSZ：Virtual memory Size，虚拟内存集
          • RSS：ReSident Size，常驻内存集
          • STAT：进程状态
            • R：running
            • S：interruptable sleeping
            • D：uninterruptable sleeping
            • T：stopped
            • Z：zombie
              -
            • +：前台进程
            • I：多线程进程
            • N：低优先级进程
            • <：高优先级进程
            • s：session leader
      • -ejf
        • -e：显示所有进程
        • -f：显示完整格式的进程信息
      • -eFH：
        • -F：显示完整格式的进程信息
        • -H：以进程层级格式显示进程相关信息
  • pgrep，pkill
    • -u uid：effective user，指明用户的所有进程
    • -U uid：real user
    • -t terminal：与指定终端相关的进程
    • -l：显示进程名
    • -a：显示完整格式的进程名
    • -P pid：显示其父进程为此处指定进程的进程列表
  • pidof： 根据进程名获取其PID号
  • top
    • 内置命令
      • 排序：
        • P：以占据的CPU百分比排序
        • M：以占据内存百分比排序
        • T：以积累占据CPU时长排序
      • 首部信息显示
        • iptime信息：l命令
        • tasks及cpu信息：t命令
        • 多颗cpu分别显示：1（数字）
        • 内存信息：m命令
      • 修改刷新时间间隔：s命令
      • 终止指定进程：k命令
      • -d：直接指明刷新时间间隔
      • -b：以批次方式显示（旧命令）
  • htop
    • s：跟踪选定进程的系统调用
    • l：显示选定进程打开的文件列表
    • a：将选定的进程绑定至某指定CPU核心
    • t：显示进程树
  • vmstat：查看虚拟内存的命令（旧命令）
    • 字段含义：
      • procs：
        • r：等待运行的进程的个数
        • b：处于不可中断睡眠态的进程的个数（被阻塞的队列的长度）
      • memory：
        • swpd：交换内存的使用总量
        • free：空闲物理内存总量
        • buffer：用于buffer的内存总量
        • cache：用于cache的内存总量
      • swap：
        • si：数据进入swap中的数据速率（kb/s）
        • so：数据离开swap中的数据速率（kb/s）
      • io：
        • bi：从块设备读入数据到系统的速率（kb/s）
        • bo：保存数据到块设备的速率（kb/s）
      • system：
        • in：interrupts，中断速率
        • cs：context switch，进程切换速率
      • cpu：
        • us：用户空间占据cpu的比例
        • sy：内核空间占据cpu的比例
        • id：空闲比例
        • wa：等待IO完成所消耗的时间比例
        • st：被虚拟化技术偷走的时间比例
    • -s显示内存的统计数据
  • pmap：报告进程的内存映射表
    • pmap [option] pid […]
    • -x：显示详细的
  • glance
    • 使用pip安装
    • -b：以Byte为单位显示网卡数据速率
    • -d：关闭磁盘I/O模块
    • -f：/path/to/simefile：设定输入文件位置
    • -o {HTML|CSV}：输出格式
    • -m：禁用mount模块
    • -n：禁用网络模块
    • -t #：延时时间间隔
    • -1：每个CPU的相关数据单独显示
    • C/S模式下运行glances命令
      • 服务端模式：
        • glances -s -B IPADDR
          • IPADDR：指明监听于本机的哪个地址
      • 客户端模式：
        • glances -c IPADDR
          • IPADDR：要连入的服务器端地址
  • dstat
    • -c：显示CPU的状态信息
    • -C #,#,…total
    • -d：显示磁盘的读写速率
    • -D：total,sda,sdb,…
    • -g：显示page相关统计数据
    • -i：显示中断的统计数据
    • -m：显示内存相关统计数据
    • -n：显示网络相关统计数据
    • -p：显示进程相关统计数据
    • -r：显示io相关统计数据
    • -s：显示swapped相关的统计数据
    • –tcp
    • –udp
    • –unix
    • –raw
    • –socket
    • –ipc
    • –top-cpu：显示最占用CPU的进程
    • –top-io：显示最占用io的进程
    • –top-mem：显示最占用内存的进程
    • –top-lantency：显示延迟最大的进程
  • kill
    • -l：显示当前系统可用信号
      • man 7 signal
      • 常用信号
        • 1) SIGHUP：无须关闭进程而让其重读配置文件
        • 2) SIGINT：中止正在运行的进程，相当于Ctrl+c
        • 9) SIGKILL：杀死正在运行的进程
        • 15) SIGTERM：终止正在运行的进程
        • 18) SIGCONT：
        • 19) SIGSTOP：
      • 指定信号的方法
        • 信号的数字标识：1
        • 信号的完整名称：SIGHUP
        • 信号的简写名称：HUP
    • killall：终止名称之下的所有进程

作业控制命令

  • 作业类型：
    • 前台作业：通过终端启动，且启动后一直占据终端
    • 后台作业：可与通过终端启动，但启动后即转入后台运行（释放终端）
  • 让作业运行于后台
    • 运行中的作业：Ctrl+z
    • 尚未启动的作业：COMMAND &
      • 注意：此类作业虽然被送往后台运行，但其依然与终端相关；如果希望送往后台后，剥离与终端的关系，使用nohub COMMAND &
  • 查看所有作业
    • jobs
  • 作业控制
    • fg [[%]JOB_NUM]：把指定的后台作业调回前台
    • bg [[%]JOB_NUM]：让送往后台的作业在后台继续运行
    • kill [%JOB_NUM]：终止指定的作业
  • 进程优先级调整
    • 进程默认启动时的nice值为0，优先级为120
    • nice -n priority pid
    • renice -n priority pid
    • 查看进程nice值
      • ps axo pid,comm,ni
  • 其他常用命令：sar、tsar、iostat、iftop
