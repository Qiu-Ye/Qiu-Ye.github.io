磁盘和文件系统管理

发表于 2018-02-28 | 分类于 系统运维 |
字数统计: 1,420 | 阅读时长 ≈ 6

磁盘管理

  • fdisk：一块硬盘最多管理15个分区
    • fdisk -l Device
    • fdisk Device
      • p：print，显示已有分区
      • n：new，创建
      • d：delete，删除
      • w：write，写入磁盘并退出
      • q：quit，放弃更改并退出
      • m：获取帮助
      • l：分区id列表
      • t：调整分区id
  • 查看内核是否已经识别新的分区
    • cat /proc/partations
  • 通知内核重新读取硬盘分区表
    • partx -a /dev/device
      • -n M:N
    • kpartx -a /dev/device
      • -f force
      • -a

文件系统管理

创建文件系统

  • 相关工具
    • mkfs.FS_TYPE /dev/device
      • ext4
      • xfs
      • btrfs
      • vfat
    • mkfs -t FS_TYPE /dev/device
      • -L ‘LABEL’：设定卷标
    • mke2fs：ext系列文件系统管理工具
      • -t {ext2|ext3|ext4}
      • -b {1024|2048|4096}
      • -L ‘LABEL’
      • -j 相当于 -t ext3
        • mkfs.ext3 = mkfs -t ext3 = mke2fs -j = mk2fs -t ext3
      • -i #：为数据空间中每多少个字节创建一个inode，此大小不应该小于block的大小
      • -N #：为数据空间创建个多少个inode
      • -m #：为管理人员预留的空间占据的百分比
      • -O FEATURE[,…]
        • -O ^FEATURE：关闭制定特性
    • mkswap：创建交换分区
      • -L ‘LABEL’
      • 前提：调整其分区的ID为82
  • 其他常用工具
    • blkid ：块设备属性信息查看
      • -U UUID：根据指定的UUID来查找对应的设备
      • -L LABEL：根据指定的LABEL来查找对应的设备
    • e2label：管理ext系列文件系统的LABEL
      • e2label device
    • tune2fs：重新设定ext系列文件系统可调整参数的值
      • -l：查看指定文件系统超级块信息
      • -L ‘LABEL’ ：修改卷标
      • -j：将ext2升级为ext3
      • -O：文件系统属性启动或禁用
      • -o：调整文件系统的默认挂载选项
      • -U UUID：修改UUID号
    • dumpe2fs：
      • -h：查看超级块信息

文件系统检测

  • fsck：file system check
    • fsck.FS_TYPE
    • fsck -t FS_TYPE
    • -a：自动修复错误
    • -r：交互式修复错误
    • FS_TYPE一定要与分区的文件系统相符
  • e2fsck：ext系列文件专用的检测修复工具
    • -y：自动回答为yes
    • -f：强制修复

文件系统挂载

  • 挂载点下原有的文件在挂载完成后会被临时隐藏
  • 通过查看/etc/mtab文件显示当前系统已挂载的所有设备
  • 通过查看/proc/mounts文件显示内核追踪到的已挂载的所有设备
  • mount [-fnrsvw] [-t vfstype] [-o options] device dir
    • device：指名要挂载的设备
      • 设备文件：/dev/sda4
      • 卷标：-L ‘LABEL’
      • UUID：-U ‘UUID’
      • 伪文件系统名称：proc，sysfs，devtmpfs，configfs
    • dir：挂载点
      • 事先存在，建议使用空目录
      • -t vsftype：指定要挂载的设备上的文件系统类型
      • -r：readonly，只读挂载
      • -w：read and write，读写挂载
      • -n：不更新/etc/mtab
      • -a：自动挂载所有支持自动挂载的设备，定义在/etc/fstab文件中，且挂载选项中中有“自动挂载”功能
      • -L ‘LABEL’：以卷标指定挂载设备
      • -U ‘UUID’：以UUID指定挂载设备
      • -B，–bind：绑定目录到另一个目录上
      • -o options：挂载文件系统的选项
        • async/sync：异步/同步模式
        • atime/noatime：是否更新访问时间戳，包含目录和文件
        • diratime/nodiratime：目录的访问时间戳
        • auto/noauto：是否支持自动挂载
        • exec/noexec：是否支持将文件系统上应用程序运行为进程
        • dev/nodev：是否支持在此文件系统上使用设备文件
        • suid/nosuid
        • remount：重新挂载
        • ro：只读
        • rw：读写
        • user/nouser：是否允许普通用户挂载此设备
        • acl：启用此文件系统上的acl功能
        • defaults：rw,suid,dev,exec,auto,nouser,and async
        • 上述选项可多个同时使用，彼此使用逗号分隔
  • 查看正在访问指定文件系统的进程：fuser -v MOUNT_POINT
  • 终止所有正在访问指定的文件系统的进程：fuser -km MOUNT_POINT
  • 文件挂载的配置文件：/etc/fstab
    • 每行定义一个要挂载的文件系统
    • 字段含义
      • 要挂载的设备或伪文件系统
        • 设备文件、LABEL、UUID、伪文件系统名称
      • 挂载点
      • 文件系统类型
      • 挂载选项
      • 转储频率
        • 0：不做备份
        • 1：每天转储
        • 2：每隔一天转储
      • 自检次序
        • 0：不自检
        • 1：首先自检，一般只有rootfs采用1

文件系统空间占用等信息的查看工具

  • df
    • -h：human-readable
    • -i：inodes instead of blocks
    • -P：以Posix兼容的格式输出
  • du：查看某目录总体空间占用状态
    • du [OPTION]… DIR
    • -h：human-readable
    • -s：summary

文件系统上的其他概念

  • Inode：Index Node，索引节点
  • inode bitmap
  • 链接文件
    • 硬链接：
      • 不能够对目录进行
      • 不能跨分区进行
      • 指向同一个inode的多个不同路径，创建文件的硬链接即为为inode创建新的引用路径，因此会增加其引用计数
    • 符号链接：
      • 可以对目录进行
      • 可以跨分区
      • 指向的是另一个文件的路径，其大小为指向的路径字符串的长度，不增加或减少目标文件的inode的引用计数
      • ln [-sv] SRC DEST
        • -s：symbolic link
        • -v：verbose
  • dd命令
    • dd if=/PATH/FROM/SRC of=/PATH/TO/EDST
      • bs=#：block size，复制单元大小
      • count=#：复制的bs数目
    • 磁盘拷贝
      • dd if=/dev/sda of=/dev/sdb
    • 备份MBR
      • dd if=/dev/sda of=/tmp/mbr.bak bs=512 count=1
    • 快速清空MBR中的bootloader
      • dd if=/dev/zero of=/dev/sda bs=256 count=1

例子

  • 创建一个20G的文件系统，块大小为2048，文件系统为ext4，卷标为TEST，要求此分区开机后自动挂载至/testing目录，且默认有acl挂载选项
    1. fdisk /dev/sda ：n +20G t 4 83 w
    2. partx -a /dev/sda
    3. mke2fs -t ext4 -b 2048 -L ‘TEST’ /dev/sda4
    4. /etc/fstab: add /dev/sda4 /testing ext4 defaults,acl 0 0
